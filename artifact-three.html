Databases

Milestone Three Enhancement

Artifact Description

The FunnelTrack I’ve created is a full stack web application that tracks the users’ behavior through an onboarding funnel and analyzes conversion rates at each stage. This application was created in May/June of 2026 as my Capstone. It includes a React frontend, a Node.js/Express backend, and a PostgreSQL database.

Justification for Inclusion

I selected this artifact for this category because the application is built around a designed relational database schema. The components that showcase my database skills are the four related tables I created, users, funnel_event, ab_test, and survey_responses. The artifact was improved by implementing a fully non redundant database schema, writing SQL queries that power the dashboard, and securing the database operations with parameterized queries to prevent SQL injections.

Course Outcomes

This enhancement addressed the course outcomes as it demonstrated the requirements asked. I used PostgreSQL and followed the best practices for schema design including normalization, primary keys, foreign keys, and appropriate data types. I also addressed the security mindset to where I implemented parameterized queries using $1, $2, and $3 placeholders instead of joining strings together. I also protected my database credentials by storing them in environment variables rather than hardcoding them in the source code.

Reflection

Building the database layer of FunnelTrack has taught me how important schema design is prior to writing a single line of code. Early on I had to think about what data I needed to store and how the tables would relate to each other. I also needed to know what queries to run to produce the analytics on my dashboard. One challenge I faced was the inconsistent naming between my code and my database. I had a table named ab_test but my code was looking for ab_tests which taught me the importance of naming my conventions carefully from the start of the project.
